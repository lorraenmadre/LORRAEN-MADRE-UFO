# Mona Lisa Stack Protocol

The Mona Lisa Stack is the daily content-to-cash and agent-routing operating system for Lorraen Madre, WISH WELL, Fruitful Frameworks, New Castle, Sanctuary Cell, Miss Honey, and the broader Universal Family Office ecosystem.

It turns wishes, stories, books, products, clips, research, and routines into repeatable daily content funnels and execution tasks.

## Core Intent

The founder creates signal, taste, approvals, story direction, and key decisions.

The agents help capture, sort, draft, package, schedule, post, track, and report.

The system should never require the founder to carry the entire machine in her nervous system. That is not leadership. That is unpaid server hosting with feelings.

## Seven Monetizable Content Paths

Each path gets one main day inside a one-hour Content Engine block.

### 1. LinkedIn Authority Path

Purpose: professional credibility, investor/funder visibility, policy/business framing, and founder signal.

Best for:

- Founder updates
- Research-backed posts
- Domestic trust insurance framing
- Universal Family Office thesis
- Startup/product progress
- Partnership and investor-facing posts

### 2. Facebook / Instagram Community Path

Purpose: relational visibility, family-facing storytelling, visual brand presence, and softer daily connection.

Best for:

- Reels
- Carousels
- Founder story posts
- Book updates
- Miss Honey lessons
- Community announcements
- Behind-the-scenes content

### 3. TikTok / Shopify Product Path

Purpose: short-form product discovery into checkout.

Best for:

- Product demos
- Sanctuary Cell products
- Book/product tie-ins
- Mini lessons
- Founder clips
- TikTok Shop and Shopify traffic

### 4. YouTube Content Path

Purpose: long-form authority, story, teaching, and clip source material.

Best for:

- Explainers
- Book readings
- WISH WELL house breakdowns
- Miss Honey lessons
- Founder updates
- Long-form source content that can be clipped later

### 5. Pinterest Discovery Path

Purpose: evergreen visual search, discovery, and long-tail traffic.

Best for:

- Pins for books
- Pins for WISH WELL templates
- Pins for Miss Honey lessons
- Pins for Sanctuary Cell products
- Pins linking to Substack, Shopify, KDP, or landing pages

### 6. Substack / Amazon KDP Publishing Path

Purpose: written authority, doctrine, subscriber relationship, and formal book publishing.

Substack builds the reader relationship and doctrine trail. Amazon KDP houses the formal book/product publication path.

Best for:

- Daily notes
- Weekly WISH WELL letters
- Book updates
- Book chapters
- KDP descriptions
- KDP keywords and categories
- Launch notes
- Reader funnel content

### 7. Google Visibility Path

Purpose: weekly search presence, discoverability, business credibility, and public indexing through Google-connected surfaces.

Best for:

- Google Business Profile posts, if applicable
- Google Sites updates
- Public Google Docs or Drive assets
- Search-friendly landing page updates
- Google Search Console checks
- Weekly proof-of-work links

## One-Hour Daily Content Engine

Daily minimum:

1. Pick one storyline.
2. Create or choose one source asset.
3. Turn it into one platform-specific output.
4. Save the asset in Drive.
5. Log the receipt in Notion / Fruitful Frameworks.
6. Create or update the execution task in Monday if needed.
7. Post to Slack only if an approval, blocker, or agent update is needed.

Suggested hour:

```text
10 minutes — choose story / product / lesson
15 minutes — create or clip source content
15 minutes — package for the day's platform
10 minutes — write caption / CTA / metadata
10 minutes — save, log, schedule, or assign
```

## First Build Order

1. Create the local Mona Lisa Stack folder.
2. Clone the Lorraen Madre UFO and New Castle repos.
3. Install local AI and coding tools.
4. Create the Mona Lisa Operator Agent prompt.
5. Build the seven role prompts.
6. Choose the workflow builder: Dify, Flowise, n8n, or CrewAI.
7. Create one test workflow: content idea to Slack approval to Monday task.
8. Only then expand to all seven content paths.

## First Agent to Build

The first agent should be the Mona Lisa Operator Agent.

Role:

Receive an idea, classify it into a content path, assign a WISH WELL house, identify the destination tool, and create the next action.

The seven specialty agents come after the Operator can route the work.
